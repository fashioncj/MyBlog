##开始
第一次做真实的需求。突然感觉自己是真·新人。

Leader说，我们的产品覆盖几亿的用户，大盘机型有3000多。所以每一行代码都得小心。

不管在哪里做事，都离不开认真。

##流程

+ 仔细评估需求
+ 方案确定
+ 组件查找+开源代码查找
+ 开发，合并，同步。
+ 是否需要添加监控，远程开关，上报开关，灰度开关等
+ 自动化测试/灰度测试
+ 兼容性报告，机型覆盖率等信息
+ Code Review
+ 上报内容确定，需要的信息+需要分析的内容
+ 合并主干
